Unison

by Patrick Sheridan
in Articles on 2015-08-05

Unison is a super useful file synchronization tool that I use every day. You can think of it like a manual and more configurable version of Dropbox or Google Drive. This has the advantage that you control when syncing happens and it's relatively easy to ignore certain files or folders. You can also sync from anywhere on your hard drive rather than having to use a designated folder as with the above options.

While there are packaged solutions using apt-get and brew, Unison is very picky about which versions are running on the computers you are trying to sync. You also need to watch which version of OCAML (a prerequisite of Unison) you are using. Since I use both OSX and Ubuntu and the package versions differ, I found it easier to install everything from source.

Installation

ETAGS (if you don't have it already)

  • sudo apt-get install exuberant-ctags

OCAML

  • wget http://caml.inria.fr/pub/distrib/ocaml-4.02/ocaml-4.02.2.tar.gz
  • tar xvzf ocaml-4.02.2.tar.gz
  • cd ocaml-4.02.2
  • ./configure -prefix /opt
  • make world.opt -j4
  • make install
  • export PATH=$PATH:/opt/bin

Unison

  • wget unison-2.48.3.tar.gz
  • tar xvzf unison-2.48.3.tar.gz
  • cd unison-2.48.3
  • make UISTYLE=text -j4
  • sudo mv unison /usr/bin/unison

Configuration

Unison configuration profiles are stored in the $HOME/.unison directory. Create a file in that folder and name it whatever you want--you will use this name when you want to synchronize using this profile. An example configuration might look like the following:

root = /Volumes/Elements/project
root = ssh://my-remote-server.com//opt/project

backup = Name *           #not strictly necessary
backuplocation = local    #these options will create backups when you sync

ignore = Name *.o
ignore = Name .DS_store
ignore = Name Thumbs.db
ignore = Name *.bin
ignore = Name build
ignore = Name {.svn}
ignore = Name *.elf
ignore = Name *.pyc
perms = 0             #file permissions not synched
